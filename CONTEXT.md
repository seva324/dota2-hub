# DotaHub Context

DotaHub 是一个面向中文用户的 Dota 2 电竞信息产品，用于查看直播比赛、赛程、赛果、战队、选手、赛事和资讯。

## Language

**比赛**:
两支战队之间的一组竞技系列，通常是 BO1、BO3 或 BO5。
_Avoid_: 指代完整系列时使用“对局”

**Series**:
用于聚合同一组 BO 系列下多张 **地图** 的数据边界，UI 上仍称为 **比赛**。
_Avoid_: 把同一 Series 下的多张地图渲染成多场独立比赛

**地图**:
一场 **比赛** 系列里的单局游戏。
_Avoid_: 当 UI 需要和阵容选择、时间线、赛后数据对齐时使用“局”

**赛事**:
包含多场 **比赛** 的锦标赛或联赛实体。
_Avoid_: 指代赛事实体时使用“赛程”

**赛程**:
按时间排列的 **比赛** 列表，包含直播中、即将开始和已结束状态。
_Avoid_: 指代日程视图时使用“赛事”

**比赛详情**:
围绕一场 **比赛** 展示比分、地图、阵容选择、经济曲线、事件、选手数据、阵容、直播源和相关比赛的页面或界面。
_Avoid_: 当界面包含直播中或进行中数据时使用“战报”

**战队速览**:
从其他界面打开的紧凑战队资料，聚焦阵容、近期状态、接下来比赛、英雄池和关键数据。
_Avoid_: 当界面是抽屉或侧滑层而非独立页面时使用“战队详情”

**选手速览**:
从其他界面打开的紧凑选手资料，聚焦身份、战队、位置、英雄池、近期状态、荣誉和最近比赛。
_Avoid_: 当界面是抽屉或侧滑层而非独立页面时使用“选手详情”

**英雄池**:
战队或选手高频使用或胜率突出的英雄集合。
_Avoid_: 当数据同时包含胜率和场次时只写“常用英雄”

## Relationships

- 一个 **赛事** 包含多场 **比赛**。
- 一个 **Series** 对应一场 **比赛**，并聚合该比赛下的一张或多张 **地图**。
- 一场 **比赛** 包含一张或多张 **地图**。
- 一个 **比赛详情** 只对应一场 **比赛**。
- 一个 **赛程** 按直播中、即将开始、已结束展示 **比赛**。
- 一个 **战队速览** 可以从 **首页**、**比赛详情** 或已结束 **比赛** 行打开。
- 一个 **选手速览** 可以从 **比赛详情** 或 **战队速览** 打开。
- 一个 **英雄池** 只归属于一个战队或一个选手。

## Example Dialogue

> **Dev:** “原型顶部同时有 `比赛` 和 `赛事`，这两个是同一个东西吗？”
> **Domain expert:** “不是。`比赛` 是看单场对阵和数据的入口；`赛事` 是锦标赛/联赛实体。首页可以同时出现两者，但不要混用。”

## Flagged Ambiguities

- 现有代码和文案把包含直播、赛程和赛果的区域称为“赛事战报”。本次 redesign 中，单场对阵使用 **比赛详情**，时间列表使用 **赛程**。
- 原型文件把战队和选手覆盖层命名为 “Flyout” 或 “Profile”。领域语言里，抽屉/侧滑层统一称为 **战队速览** 和 **选手速览**，把 **详情** 留给未来独立页面。
- 现有比赛数据可能以地图/单局为单位返回。重构后的首页和比赛详情必须以 **Series** 为聚合边界，把同一 Series 下的地图展示为 `地图 1 / 地图 2 / 地图 3`，而不是拆成多张比赛卡。
