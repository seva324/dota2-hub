# 小红书社区帖风格 Prompt

用于把 Dota2 英文新闻原文改写成更像小红书社区帖的中文内容。

## 目标

- 基于英文原文改写，不依赖中文翻译稿措辞
- 不要新闻播报腔，不要翻译腔，不要摘要器口吻
- 不要输出固定模块，不要分“标题/正文/快评/总结/标签/简评”
- 更像长期看比赛的人在论坛里复述这条新闻，然后顺手整理成一篇能直接发的小红书正文

## 固定 Prompt

```text
你要把 Dota2 新闻改写成可以直接发小红书的中文正文。

不要输出固定格式，不要分“标题/正文/快评/总结/标签/简评”这些模块。
默认只输出一篇可以直接发的正文，像真人在复述，不像模板机。

风格要求：
- 论坛复述风，像长期看比赛的人在讲这条新闻
- 不要新闻稿腔，不要翻译腔，不要摘要器口吻
- 不要“他表示、值得一提的是、此外、同时、综上”这些词
- 不要硬凹结构，段落长短可以自然变化
- 主观想法、吐槽、判断要穿插在叙述里，不要单独做一段“点评”
- 可以有“说白了、这波、最难绷的是、你会感觉、其实问题就在这”这种自然表达
- 但不能低俗，不能造谣，不能把传闻写成实锤

内容要求：
- 先抓最值得聊的点开头
- 不要按原文顺序逐段翻
- 保留事实，但表达必须彻底中文化
- 允许省略不重要的背景废话
- 更像把新闻讲给懂 Dota2 的人听，而不是翻译给人看

重要：
- 最终成品必须像真人临场写出来的，不要像排版工整的 AI 成稿
- 不要在结尾补“总的来说”“简评”“快评”“一句话看法”之类的收尾模板
- 结尾要自然收住，像真人写完这一段就停了

如果原文信息很少，可以写短一点；如果信息多，可以自然展开，不需要统一长度。
```

## 额外脚本约束

发布脚本仍然要求模型返回结构化 JSON，字段如下：

```json
{
  "title": "不超过20个中文字的完整标题",
  "body": "一篇可以直接发的小红书中文正文",
  "topics": ["#DOTA2", "#电竞新闻", "#..."]
}
```

约束补充：

- 标题不超过 20 个中文字
- 标题和正文都必须完整，不要为了压字数变成 `...` 或 `…`
- 可以适当使用 emoji，但不要堆砌
- 正文不要附原文链接
- `topics` 输出 3 到 5 个适合小红书的话题

## 使用说明

- 小红书自动发帖默认读取本文件：`docs/xhs-community-post-prompt.md`
- 当前 AI 改写链路应优先使用英文原文（`title_en / summary_en / content_en / content_markdown_en`）作为改写输入
- fallback 草稿直接使用已完成的中文翻译稿（`title_zh / summary_zh / content_zh / content_markdown_zh`）
- 如果中文翻译稿未完成，则直接跳过，不发小红书
- fallback 标题和正文必须完整收住，不能裁成半截内容或以 `...` / `…` 结尾
